Complete schema reference for `app.yaml` and `domain.yaml`.

## `app.yaml`

### Complete Example

```yaml
name: my_app
description: "My application"

database:
  dsn: postgres://user:pass@host:5432/     # full DSN (overrides fields below)
  host: localhost                            # or use individual fields
  port: 5432
  user: myapp_user
  password: ${DB_PASSWORD}
  ssl_mode: disable
  max_connections: 10
  min_connections: 2
  connect_timeout: 5s

auth:
  domain: company                # optional — shared user pool
  session_expiry: 24h            # Go duration, default: 24h
  allow_registration: true       # default: true

keys:
  publishable_key: vK8mN2pXqR5tL9wA3hE6yJ4cF7uB1nD0

secrets:
  STRIPE_KEY: ${STRIPE_KEY}
  SENDGRID_KEY: ${SENDGRID_KEY}

storage:
  endpoint: https://s3.amazonaws.com
  bucket: my-app-bucket
  region: us-east-1
  access_key_id: ${S3_ACCESS_KEY}
  secret_access_key: ${S3_SECRET_KEY}
  presign_expiry: 1h

jobs:
  max_attempts: 3
  timeout: 15s
  custom:
    _send_email:
      max_attempts: 5
    _generate_report:
      timeout: 1h
      max_attempts: 1

cron:
  - schedule: "0 3 * * *"
    function: _cleanup_expired
  - schedule: "*/10 * * * *"
    function: _process_pending
    payload:
      mode: full

policies:
  posts:
    select:
      expression: "true"
      columns: ["*"]
    insert:
      expression: "auth.authenticated"
      columns: ["title", "body"]
      defaults:
        user_id: "auth.uid"
        created_at: "now()"
    update:
      expression: "auth.uid == row.user_id"
      columns: ["title", "body"]
      defaults:
        updated_at: "now()"
    delete:
      expression: "auth.uid == row.user_id"
      soft: deleted_at
```

### Field Reference

#### Top-level

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | Yes | App identifier. Lowercase alphanumeric + underscores, starts with letter. |
| `description` | string | No | Human-readable description. |

#### `database`

All fields optional. If absent, inherits `DATABASE_URL` environment variable.

| Field | Type | Default | Description |
|---|---|---|---|
| `dsn` | string | — | Full PostgreSQL DSN (overrides all other fields) |
| `host` | string | — | PostgreSQL host |
| `port` | int | `5432` | PostgreSQL port |
| `user` | string | — | Database user |
| `password` | string | — | Database password (supports `${VAR}`) |
| `ssl_mode` | string | `disable` | SSL mode: `disable`, `require`, `verify-ca`, `verify-full` |
| `max_connections` | int | `10` | Maximum pool connections |
| `min_connections` | int | `2` | Minimum pool connections |
| `connect_timeout` | duration | `5s` | Connection timeout |

**Resolution order:** `dsn` → `host` fields → `DATABASE_URL` env var → error.

#### `auth`

| Field | Type | Default | Description |
|---|---|---|---|
| `domain` | string | — | Domain name for shared auth. Omit for local `_users`. |
| `session_expiry` | duration | `24h` | Session token lifetime |
| `allow_registration` | bool | `true` | Allow new user registration |

#### `keys`

| Field | Type | Required | Description |
|---|---|---|---|
| `publishable_key` | string | Yes | 32-byte base64url key. Safe to commit. Generated by `backd bootstrap`. |

The `secret_key` is never in `app.yaml` — it is generated on first startup and stored in `_secrets`.

#### `secrets`

Map of `name: ${ENV_VAR}`. Values must use `${VAR}` syntax.

#### `storage`

Entire block optional. Omit to disable storage.

| Field | Type | Default | Description |
|---|---|---|---|
| `endpoint` | string | — | S3-compatible endpoint URL |
| `bucket` | string | — | S3 bucket name |
| `region` | string | — | AWS region |
| `access_key_id` | string | — | S3 access key (supports `${VAR}`) |
| `secret_access_key` | string | — | S3 secret key (supports `${VAR}`) |
| `presign_expiry` | duration | `1h` | Presigned URL expiry for secure files |

#### `jobs`

| Field | Type | Default | Description |
|---|---|---|---|
| `max_attempts` | int | `3` | Default max retry attempts |
| `timeout` | duration | `15s` | Default execution timeout |
| `custom` | map | — | Per-function overrides (same fields) |

#### `cron`

Array of scheduled jobs.

| Field | Type | Required | Description |
|---|---|---|---|
| `schedule` | string | Yes | 5-field cron expression: `minute hour day month weekday` |
| `function` | string | Yes | Function directory name |
| `payload` | map | No | Static JSON payload passed to the function |

#### `policies`

Map of `table_name` → `operation` → policy entry. See [Policies]({{< ref "/guides/policies" >}}).

| Field | Operations | Required | Description |
|---|---|---|---|
| `expression` | All | Yes | CEL boolean expression |
| `check` | INSERT, UPDATE | No | Additional write guard |
| `columns` | SELECT, INSERT, UPDATE | No | Column allowlist. `["*"]` = all |
| `defaults` | INSERT, UPDATE, DELETE (soft) | No | Auto-applied values |
| `soft` | DELETE | No | Column name for soft deletion |

## `domain.yaml`

```yaml
name: company
description: "Company-wide user pool"
provider: password
session_expiry: 24h
allow_registration: true

database:
  # same fields as app database config
```

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | Yes | Domain identifier. Same naming rules as apps. |
| `description` | string | No | Human-readable description |
| `provider` | string | Yes | Auth provider. Currently: `password` only. |
| `session_expiry` | duration | `24h` | Session lifetime |
| `allow_registration` | bool | `true` | Allow new user registration |
| `database` | object | No | Same fields as app `database` config |

## Environment Variable Substitution

Values using `${VAR_NAME}` syntax are resolved from environment variables at config load time:

```yaml
secrets:
  STRIPE_KEY: ${STRIPE_KEY}     # resolved from env
storage:
  access_key_id: ${S3_KEY}      # resolved from env
database:
  password: ${DB_PASSWORD}      # resolved from env
```

If a referenced variable is not set, `backd validate --check-env` reports the error. The `api secrets apply` command validates all references before writing.
